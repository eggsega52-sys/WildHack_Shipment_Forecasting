# WildHack: Прогнозирование объемов отгрузок

Решение задачи по прогнозированию 30-минутных и часовых объемов отгрузок для 1000 логистических маршрутов. 

## Основные инновации решения:
*   **Vise Method (Метод Тисков):** Математически точное восстановление 30-минутных интервалов отгрузки из часовых окон.
*   **Ridge Legion:** Ансамбль из 1000 локальных Ridge-экспертов для стабилизации нейросетевых прогнозов.
*   **Triple Window Consistency:** Физический фильтр, гарантирующий соблюдение закона накопления массы ($T_t + T_{t+2} \ge T_{t+1}$).
*   **Dissensus Weighting:** Динамическое управление весами ансамбля на основе неопределенности (несогласия) моделей.

## Структура проекта
Для получения финального решения необходимо запустить ноутбуки в следующем порядке:

### 1. Подготовка данных и базовых моделей
*   `017_AutoGluon_Specialized_Peak_Hours.ipynb` — Обучение модели на утренних якорях.
*   `028_Transformer_Global_Pulse_72h.ipynb` — Обучение Transformer с глобальным контекстом склада.
*   `038_AutoGluon_Best_Quality_Heavy.ipynb` — Стэкинг тяжелых моделей (Chronos/TFT) на часовых данных.
*   `081_AutoGluon_30min_Best_Quality.ipynb` — Стэкинг моделей на 30-минутных интервалах.
*   `105_Stable_Legion_Ridge_Experts.ipynb` — Обучение 1000 локальных линейных экспертов.

### 2. Финальный ансамбль и постобработка
*   `108_Strategy3_Dynamic_Dissensus.ipynb` — Реализация логики динамических весов и фильтра Triple Window.
*   `113_Final_Precision_Strike.ipynb` — **ГЛАВНЫЙ ФАЙЛ.** Сборка финального бленда и калибровка целевого объема (Bias Correction).

## Инструкция по воспроизведению
1. Разместите исходные данные в папку `data/` (`train_solo_track.parquet`, `test_solo_track.parquet`).
2. Установите зависимости: `pip install -r requirements.txt`.
3. Последовательно запустите ноутбуки. Итоговые файлы будут сохранены в `submissions/`.
